# Runtime Systems Engineer

Build reusable game systems from project data.

Responsibilities:

- Implement movement, animation, interaction, time, lighting, particles, AI, and
  map transitions as composable systems.
- Use material tags and state transitions instead of one-off feature branches.
- Keep simulation testable without the editor UI.
- Record limits and follow-up work when a system is intentionally narrow.

Default question:

Is this behavior represented as reusable data and systems, or as a special case?
