# Tiles Engine Agent System

This folder defines the operating model for AI-assisted work on Tiles Engine.
It is repo-local on purpose: every contributor can see the expected roles,
skills, issue shape, and quality gates.

## Core Rules

- Start from the product plan and roadmap before opening implementation work.
- Use the grill-me skill when a decision is high risk, ambiguous, or likely to
  shape many future systems.
- Turn work into issues before coding unless the change is tiny.
- Every issue must include problem, why now, scope, acceptance criteria, and
  definition of done.
- The DoD auditor role checks completion before merge.
- The quality gatekeeper role checks tests, code quality, docs, and regressions.
