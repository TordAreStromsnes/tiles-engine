# Licensing

## Code License

Tiles Engine source code, documentation, schemas, tests, configuration, and
repo-native tooling are licensed under either:

- MIT
- Apache-2.0

at the user's option.

The canonical license files are:

- [../LICENSE](../LICENSE)
- [../LICENSE-MIT](../LICENSE-MIT)
- [../LICENSE-APACHE](../LICENSE-APACHE)

Rust crates use the SPDX expression `MIT OR Apache-2.0`.

## Contribution Boundary

Unless a contributor explicitly marks a submission otherwise, any intentional
contribution to source code, documentation, schemas, tests, configuration, or
repo-native tooling is submitted under `MIT OR Apache-2.0`.

The project does not require a separate contributor license agreement at this
stage.

## Asset Boundary

Sprites, textures, audio, fonts, sample packs, project templates, exported
worlds, and demo projects are treated as assets, not automatically as source
code.

Asset contributions must include an explicit asset license. Preferred defaults:

- `CC0-1.0` for reusable starter assets.
- `CC-BY-4.0` for assets that require attribution.

Third-party assets must include license and attribution notes, and their license
must permit redistribution and modification in this repository.

User-created assets in a local Tiles Engine project remain owned by their
creator unless they choose to publish or share them under a separate license.

## Future Online Sharing

When community asset sharing is designed, each shared asset pack should carry:

- Asset license.
- Attribution metadata.
- Source/provenance notes.
- Compatibility/version metadata.

The engine code license does not decide the license of user-created games,
assets, or worlds.
